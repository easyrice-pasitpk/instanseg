InstanSeg inference class

class instanseg.inference_class.InstanSeg(model_type: str | Module = 'brightfield_nuclei', device: str | None = None, image_reader: str = 'tiffslide', verbosity: int = 1)

Bases: object

Main class for running InstanSeg.

Parameters:

  • model_type – The type of model to use. If a string is provided, the model will be downloaded. If the model is not public, it will look for a model in your bioimageio folder. If an nn.Module is provided, this model will be used.

  • device – The device to run the model on. If None, the device will be chosen automatically.

  • image_reader – The image reader to use. Options are “tiffslide”, “skimage.io”, “bioio”, “AICSImageIO”.

  • verbosity – The verbosity level. 0 is silent, 1 is normal, 2 is verbose.

eval_medium_image(image: Tensor, pixel_size: float | None = None, normalise: bool = True, tile_size: int = 512, batch_size: int = 1, return_image_tensor: bool = True, normalisation_subsampling_factor: int = 1, target: str = 'all_outputs', rescale_output: bool = True, **kwargs) → Tensor | Tuple[Tensor, Tensor]

Evaluate a medium input image using the InstanSeg model. The image will be split into tiles, and then inference and object merging will be handled internally.

Parameters:

  • image: – The input image(s) to be evaluated.

  • pixel_size – The pixel size of the image, in microns. If not provided, it will be read from the image metadata.

  • normalise – Controls whether the image is normalised.

  • tile_size – The width/height of the tiles that the image will be split into.

  • batch_size – The number of tiles to be run simultaneously.

  • return_image_tensor – Controls whether the input image is returned as part of the output.

  • normalisation_subsampling_factor – The subsampling or downsample factor at which to calculate normalisation parameters.

  • target – Controls what type of output is given, usually “all_outputs”, “nuclei”, or “cells”.

  • rescale_output – Controls whether the outputs should be rescaled to the same coordinate space as the input (useful if the pixel size is different to that of the InstanSeg model being used).

  • kwargs – Passed to pytorch.

Returns:

A tensor corresponding to the output targets specified, as well as the input image if requested.

eval_whole_slide_image(image: str, pixel_size: float | None = None, normalise: bool = True, normalisation_subsampling_factor: int = 1, tile_size: int = 512, overlap: int = 80, detection_size: int = 20, save_geojson: bool = False, use_otsu_threshold: bool = False, **kwargs)

Evaluate a whole slide input image using the InstanSeg model. This function uses slideio to read an image and then segments it using the instanseg model. The segmentation is done in a tiled manner to avoid memory issues.

Parameters:

  • image – The input image to be evaluated.

  • pixel_size – The pixel size of the image, in microns. If not provided, it will be read from the image metadata.

  • normalise – Controls whether the image is normalised.

  • tile_size – The width/height of the tiles that the image will be split into.

  • overlap – The overlap (in pixels) betwene tiles.

  • detection_size – The expected maximum size of detection objects.

  • batch_size – The number of tiles to be run simultaneously.

  • normalisation_subsampling_factor – The subsampling or downsample factor at which to calculate normalisation parameters.

  • use_otsu_threshold – bool = False. Whether to use an otsu threshold on the image thumbnail to find the tissue region.

  • kwargs – Passed to pytorch.

Returns:

Returns a zarr file with the segmentation. The zarr file is saved in the same directory as the image with the same name but with the extension .zarr.
